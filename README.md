# Ветки в клетке

Логическая игра про станции московского метро: игрок заполняет сетку 3x3 станциями, каждая ячейка должна одновременно подходить под условие строки и условие столбца.

Проект задуман как московский аналог Tubedoku/Immaculate Grid: короткая ежедневная загадка, проверка знаний метро и аккуратная работа с открытыми данными.

## Быстрый старт

```bash
python3 scripts/build_game_data.py --source ../moscow-metro-doku-data/data/processed/stations_base_with_wikipedia_map.csv
python3 -m http.server 8000
```

После запуска открой:

```text
http://localhost:8000
```

## Что уже есть

- `index.html` - первая версия игрового экрана.
- `src/app.js` - логика загрузки данных, генерации дневной сетки, поиска станций и проверки ответов.
- `src/styles.css` - визуальный слой без внешних зависимостей.
- `scripts/build_game_data.py` - сборка игрового JSON из CSV с данными метро.
- `public/data/game-data.json` - генерируемый файл с нормализованными станциями и наборами подсказок.
- `docs/roadmap.md` - подробный план реализации от прототипа до релиза.

## Игровая идея

В каждой загадке есть три условия по строкам и три условия по столбцам. Например:

- строка: `Сокольническая линия`
- столбец: `открыта до 1950 года`
- правильная ячейка: любая станция Сокольнической линии, открытая до 1950 года.

Игрок не выбирает из списка фиксированных вариантов, а вводит название станции. Одна станция может использоваться только один раз в сетке.

## Данные

На старте проект использует CSV из соседнего репозитория:

```text
../moscow-metro-doku-data/data/processed/stations_base_with_wikipedia_map.csv
```

Скрипт сборки оставляет только пригодные для игры поля, добавляет производные признаки и формирует категории подсказок. Позже этот слой стоит превратить в более строгий датасет с тестами качества.
